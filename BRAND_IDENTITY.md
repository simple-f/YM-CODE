## 📁 存储结构设计

### 目录命名规范

```
ym-code/                    # 项目根目录（短横线连接）
├── core/                   # 核心引擎（简洁命名）
├── skills/                 # 技能系统（直观命名）
├── tools/                  # 工具集合（清晰分类）
├── memory/                 # 记忆系统（易于理解）
├── cli/                    # 命令行界面（行业惯例）
├── extensions/             # 扩展系统（开放态度）
├── config/                 # 配置文件（集中管理）
├── logs/                   # 日志文件（标准命名）
└── workspace/              # 工作空间（用户数据）
```

### 文件命名规范

**Python 文件：** `snake_case.py`
```
skill_registry.py           # 技能注册表
code_analyzer.py            # 代码分析器
update_checker.py           # 更新检查器
```

**配置文件：** `kebab-case.json/yaml`
```
ym-config.yaml              # 主配置文件
skills-config.yaml          # 技能配置
mcp-servers.yaml            # MCP 服务器配置
```

**文档文件：** `UPPER_CASE.md`（重要文档）
```
README.md                   # 项目说明
QUICKSTART.md               # 快速开始
BRAND_IDENTITY.md           # 品牌身份（本文件）
```

```
│   ├── sessions/           # 会话记录
│   └── knowledge/          # 知识库
├── cache/                  # 缓存数据
│   ├── updates/            # 更新缓存
│   └── metrics/            # 指标数据

    └── ym-code.log

```

```
│   └── symbols.json        # 符号索引
```
