# YM-CODE 对话使用指南

**更新时间：** 2026-03-19

---

## 🎯 两种方式

### 方式 1：单个 Agent 对话

**适用场景：**
- 有明确的问题需要某个专业 Agent 解答
- 想要快速得到答案
- 不需要多角色讨论

**使用步骤：**
1. 访问 http://localhost:8080/web/agents.html
2. 点击选择一个 Agent 卡片（如：架构师）
3. 点击 **"💬 单个对话"** 按钮
4. 在对话页面输入问题，发送

**示例：**
```
问架构师：帮我设计一个电商系统的架构
问后端开发：帮我写一个用户登录 API
问测试工程师：帮我 review 这段代码
```

---

### 方式 2：多 Agent 协作

**适用场景：**
- 复杂问题需要多个角色讨论
- 需要不同角度的建议
- 想要看到 Agent 之间的协作过程

**使用步骤：**
1. 访问 http://localhost:8080/web/agents.html
2. 选择多个 Agent 卡片（如：架构师 + 后端 + 前端 + 测试）
3. 点击 **"👥 多 Agent 协作"** 按钮
4. 在对话页面输入问题，发送
5. Agent 们会依次回复，自动协作

**示例：**
```
问：帮我开发一个博客系统

架构师：设计整体架构...
后端开发：实现 API...
前端开发：实现界面...
测试工程师：编写测试...
```

## 💡 使用技巧

### 单个对话技巧
- **明确角色**：选择对应专长的 Agent
- **问题具体**：描述清楚你的需求
- **追问**：可以继续追问细节

### 多 Agent 协作技巧
- **选择相关 Agent**：不要选太多（3-5 个为宜）
- **问题开放**：让不同 Agent 有发挥空间
- **等待回复**：Agent 会依次发言

---

## 🔧 故障排查

### 看不到 Agent 列表
1. 检查服务是否运行：`python start_dashboard.py`
2. 刷新页面（Ctrl + F5）
3. 检查 `configs/team.json` 是否存在

### 对话无响应
1. 检查 API Key 是否配置
2. 查看浏览器控制台错误
3. 查看服务器日志
