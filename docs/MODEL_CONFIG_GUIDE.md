## ⚙️ 配置说明

### 环境变量配置

所有模型都在**阿里云百炼（codingplan）**下，只需配置一个 API Key：

```bash
# 阿里云百炼（通义千问）- 统一接口
OPENAI_API_KEY=sk-xxx
OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1

# 可选：备用配置
DASHSCOPE_API_KEY=sk-xxx
```

**所有 Agent 共用同一个 API Key 和接口！**

### Agent 配置参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `temperature` | 创造性（0-1） | 架构师 0.7，开发 0.5，测试 0.3 |
| `max_tokens` | 最大输出长度 | 4096（复杂任务 8192） |
| `system_prompt` | 角色设定 | 根据职责定制 |

## 🚀 使用场景

### 1. 新项目开发

**推荐 Agent：** 架构师 + 后端 + 前端 + 测试

```
工作流程：
1. 架构师设计整体架构
2. 后端开发 API 和数据库
3. 前端开发界面
4. 测试工程师审查代码
```

### 2. 代码审查

**推荐 Agent：** 测试工程师 + 对应开发

```
工作流程：
1. 测试工程师审查代码
2. 开发工程师修复问题
3. 测试工程师再次审查
```

### 3. 技术方案咨询

**推荐 Agent：** 技术顾问 + 架构师

```
工作流程：
1. 技术顾问分析需求
2. 架构师设计方案
3. 技术顾问评估风险
```

### 4. 快速原型

**推荐 Agent：** 全栈开发 + 前端开发

```
工作流程：
1. 全栈开发搭建框架
2. 前端开发实现界面
3. 快速迭代优化
```

## 🔧 自定义配置

### 添加新 Agent

在 `configs/team.json` 中添加：

```json
{
  "id": "ai8",
  "name": "数据工程师",
  "role": "developer",
  "model": "qwen3.5-plus",
  "temperature": 0.5,
  "system_prompt": "你是一位数据工程师，擅长...",
  "capabilities": ["data", "etl", "analytics"]
}
```

### 修改现有 Agent

```json
{
  "agents": [
    {
      "id": "ai1",
      "temperature": 0.8,  // 修改温度
      "max_tokens": 8192   // 修改最大长度
    }
  ]
}
```

### 创建工作流

```json
{
  "workflow": {
    "data_project": ["ai1", "ai2", "ai8"],
    "ai_project": ["ai1", "ai2", "ai3", "ai6"]
  }
}
```

## 💡 最佳实践

### 1. 模型选择原则

- **代码生成** → qwen3-coder-plus / qwen3-coder-next
- **架构设计** → qwen3.5-plus / qwen3-max
- **代码审查** → kimi-k2.5
- **技术咨询** → qwen3-max / qwen3.5-plus
- **产品设计** → MiniMax-M2.5
- **全栈开发** → glm-5

### 2. 温度设置

- **创造性任务**（设计、创意）→ 0.7-0.9
- **编码任务**（开发、实现）→ 0.5-0.7
- **审查任务**（测试、审查）→ 0.2-0.4

### 3. 成本优化

- 简单任务使用轻量模型（coder-next）
- 复杂任务使用强大模型（max、plus）
- 批量任务使用批量 API
